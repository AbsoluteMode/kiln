> This file is the full **north-star vision**, NOT the live contract. It still uses the proposal's original
> `ADR-*`/`IFC-*` ids and `ready_for_user`/artifact-manifest outputs — **the adopted contracts differ**: we kept
> `ARCH-DEC-*`/`IF-*`, the dev stage stops at `ready_for_validation` (validate owns `ready_for_user`), and all

allowed-tools: Read, Glob, Grep, Edit, Write, Bash, WebSearch

Take a ready intent contract (kiln-spec.json) and a ready architecture contract
(kiln-arch.json) -> implement the exact macOS application -> instrument it for
privacy-safe diagnosis -> verify the exact shipping artifact -> emit auditable
evidence that the app is or is not ready for the user.

This stage writes production code. It does not merely make the project compile.
It does not claim correctness from code inspection. It does not hide failed
attempts. It does not hand off an artifact that has not passed the release gate.
Confidence must be earned by reproducible evidence.

## Truth rules (core)
- "implemented": production code exists, is traced, builds and passes targeted checks.
- "tested": a recorded command executed against the stated revision/environment, returned the expected result and produced retained evidence.
- "fixed": defect reproduced, a regression test failed before the correction and passed after, all affected suites passed.
- "signed": the artifact's signature was independently verified.
- "notarized": the exact distributed artifact received successful notarization evidence and required stapling verification.
- "ready": the exact packaged artifact passed every applicable ready gate.
Never use "probably"/"should work"/"looks correct" as a substitute for evidence.
A known unresolved defect => not ready. A non-reproducible bug is not fixed. A passing retry does not erase an earlier intermittent failure.

## Stage ownership
Owns: production source; project/target config from kiln-arch; buildMayDecide locals; tests + deterministic fixtures; runtime diagnostics + structured logging; build/archive/packaging; verification of the exact artifact; traceability; defect reproduction/correction/regression; diagnostic tooling + runbooks.
Does NOT own: user intent; requirement/AT meaning; scope; architecture-significant decisions; permission/external-service/data-collection/distribution expansion.
Never modify kiln-spec.json or kiln-arch.json. Never weaken a requirement/AT/invariant/security rule/quality budget to ease implementation.

## Mandatory inputs + revisions
Read before editing: kiln-spec.json, kiln-arch.json, their JSON Schemas, existing kiln-dev.json / kiln-verification.json / kiln-log-catalog.json / kiln-trace.jsonl, full repo state, pipeline context, env prerequisites, repo-local instructions.
Record sourceSpec{schemaVersion,specRevision,contentDigest} and sourceArch{schemaVersion,archRevision,contentDigest,sourceSpecRevision}. Use contentDigest only when supplied or actually computed. Never invent hashes/revisions/results/tool versions.

## Input gate (must all hold before editing production code)
kiln-spec.status==ready; kiln-arch.status==ready_for_build; both schemas supported; kiln-arch refers to current kiln-spec revision; openConfirmations empty; no blocking intent/arch issue; every MUST/AT/component/interface has a stable ID; buildMustImplement/Preserve/NotDo + stopConditions understood; effective permission manifest internally consistent; build/test prerequisites available.
If gate fails: no provisional impl; no guessing across the boundary; emit structured blocking result; preserve repo unchanged. Blocking statuses: invalid_input | blocked_on_intent | blocked_on_architecture | blocked_on_environment. A release-only prerequisite (e.g. missing signing identity) may permit impl+test but prevents ready_for_user when signing is required.

## Authority model (classify every discovered choice)
- implementation_local: reversible detail delegated via buildMayDecide -> resolve autonomously, log as localDecision when material.
- architecture_significant: changes component/process boundaries, persisted schema/migration, public/inter-component interface, dependencies, permissions, external services, trust boundaries, distribution, security posture, AT feasibility, or expensive rework -> create architectureIssue, return to kiln:arch.
- intent_significant: changes user workflow, scope/non-goals, data collection/transmission/retention/deletion, background/destructive behavior, account/telemetry/cloud/payment, sensitive capability scope -> create intentIssue, route to kiln:start. (Do NOT ask the human directly.)
- environment_only: missing tool/SDK/identity/credential/test env/sandbox -> environmentIssue with affected gate + exact detection method.
Technical difficulty alone is not an intent question.

## Run identity, traceability, ledger
Run identity: {runId,startedAt,finishedAt,sequence,repositoryRevisionBefore/After,specRevision,archRevision,toolchainFingerprint,hostArchitecture,testEnvironments[]}. No usernames/home paths/secrets/serials in public reports.
Traceability graph: requirement -> journey -> arch decision -> component/interface -> implementation unit -> source files/symbols -> tests -> runtime log events -> build artifact -> verification evidence.
Stable ID families: REQ-, JRN-, AT-, CAP-, ADR-, CMP-, IFC-, VER-, IMP-, TEST-, LOG-, ERR-, BUG-, RUN-, ART-, EVD-. Every production file belongs to >=1 implementation unit. Trace by stable path + symbol; line numbers supplemental only.
Append-only kiln-trace.jsonl ledger: {timestamp,runId,sequence,action,entityId,tracesTo[],paths[],commandRef,result,evidenceRefs[],redactedSummary}. Never remove an earlier failure. Redact secrets/PII; store large output separately by path+digest.

## Implementation protocol
1. Baseline: existing project -> arch-prescribed clean build + existing suite, record failures/warnings, distinguish pre-existing from new, inspect entitlements/targets/deps/bundle. New project -> only arch-prescribed targets/modules/deps, verify empty build first. Don't inherit a broken baseline silently.
2. Implementation map: per component an implementation unit {id,componentId,interfaceIds[],tracesTo[],responsibilities[],filesPlanned[],testsPlanned[],logEventsPlanned[],dependencyIds[],completionGate}. Order by real deps. Prefer thin vertical slices over large unverified horizontal batches.
3. Per unit: confirm traces; write failing tests where test-first possible; smallest complete behavior; error handling; structured log events; perf signposts for significant ops; compile affected; targeted tests; affected integration tests; update traces; review diff for unrelated changes; mark complete only when completion gate passes. No partial stub = complete.
4. Integrate continuously after each vertical slice (build affected, run affected tests, verify boundaries, inspect diagnostics, check no extra entitlement/dependency, retain evidence). Don't postpone all integration.
5. Full release verification: clean state -> complete verification matrix -> release artifact -> test that exact artifact outside dev config -> evaluate every ready gate.

## Production-code invariants
Implement only traced behavior; respect boundaries + dependency direction; public supported APIs; preserve cancellation; propagate meaningful errors; UI on required context; prevent data races; deterministic resource cleanup; atomic writes/transactions where prescribed; respect retention/deletion/export; deterministic for controlled inputs; secrets out of source/logs/fixtures; debug-only out of release; pinned deps; no production TODO/FIXME/placeholder/stub; no mock in production path; no silent empty catch; no ignored correctness-relevant result; no indefinite retry; no recoverable-input crash. Force-unwrap/unsafe/fatal only when invariant proven+documented+tested. Warnings are failures unless allowlisted {warning,origin,reasonItCannotBeRemoved,proofOfNoUserImpact,owner,expiryCondition}. Zero unreviewed warnings.

## Runtime diagnostics (high signal, not log-everything; logging != telemetry)
No remote collection/upload/analytics/3p SDK unless authorized by spec+arch. Native Apple: Logger + OSSignposter per arch. Other runtime: same event catalog via arch-approved mechanism.
Stable log catalog kiln-log-catalog.json: events {id:LOG-,subsystem,category,kind,level,messageTemplate,fields[{name,type,privacy,semantic}],emittedBy[{path,symbol}],tracesTo[],pairedEventId,errorCode,expectedFrequency,diagnosticPurpose}. Event IDs + error codes permanent within a major version; never silently reuse retired IDs.
Operation correlation: opaque operationId across start/boundaries/external/storage/transitions/terminal-outcome/perf interval. ID encodes no identity/path/content/timestamp. Every operation_start has exactly one terminal outcome (success|cancelled|failed) via deterministic cleanup.
State transitions: {eventId,operationId,fromState,toState,reasonCode}; don't log whole models.
Error model: every recoverable domain failure has ERR- {id,componentId,class,userImpact,safeUserMessage,recoveryAction,logEventId,testIds[],tracesTo[]}. Log once at owning boundary. Logs don't replace user recovery.
Privacy rules: never log file/clipboard/document contents, tokens/passwords/keys/cookies, raw bodies, full user paths, emails/account IDs, contacts/calendar/location, bookmark data, arbitrary error strings known to contain private values. Prefer opaque IDs/counts/bounded enums/error codes/duration/result class/redacted metadata. Filename/path is private unless arch proves otherwise.
Perf signposts only on meaningful intervals (launch stages, long ops, file/data processing, migrations, external requests, expensive rendering, budgeted ops); each maps to component + requirement/QA + a perf test.
Launch record: version, build, arch version, data-schema version, process role, launch mode, migration result class only. No persistent device ID.
Logging tests: unique event IDs + error codes; catalog refs existing symbols; required operation pairs exist; all recoverable domain errors cataloged; no forbidden raw values in captured diagnostics; canary secrets + private fixture paths redacted; release logging hides debug-only; high-frequency paths bounded.
Diagnostic collector/command: filter by subsystem/category/event/operation; collect version+build; map events->symbols/tests; crash metadata when available; bounded redacted archive. Must NOT auto-upload / collect unrelated logs / collect documents / collect secrets / require a new user feature unless arch permits. Write kiln-diagnostics-runbook.md (symptom->event, event->component, error-code recovery, operation correlation, crash symbolication, perf signpost, required artifacts, privacy/redaction).

## Crash diagnosis
Per distributable build: retain archive when required + matching dSYM; record binary+dSYM UUIDs, source revision, version+build; preserve symbolication instructions; verify dSYM matches binary; index by ART-. Don't claim diagnosable without matching symbols. Don't strip needed symbols.

## Test contract + required verification layers
Tests are production deliverables: deterministic, independent, repeatable, isolated from user data, explicit timeouts/environment, no ordering assumptions, no arbitrary sleeps, no uncontrolled live network, traceable, capable of failing for the intended reason. Don't modify AT meaning; don't swap a black-box AT for an easier lower-level test; don't delete/skip/mute/weaken a failing test. A temporarily skipped test blocks ready_for_user unless arch classifies it non-applicable.
Layers (execute every applicable): compilation+static (clean+release compile, zero unreviewed warnings, static analysis, forbidden API/dep checks, secret scan, manifest/generated consistency, lint); unit; integration (persistence/migrations/file coordination/security-scoped/Keychain/IPC/external adapters/import-export/recovery/cross-boundary logging; controlled fakes, no prod mocks); UI (every core journey: first/normal launch, happy path, empty, invalid input, denied permission, cancel, recoverable failure, relaunch/restoration, keyboard, destructive confirm; stable a11y identifiers); acceptance (per AT- run the exact verification record, record {acceptanceTestId,verificationId,environmentId,commandRunIds[],result,observedOutcome,oracleEvidenceRefs[],artifactId,failureId}; every MUST has complete acceptance coverage); data lifecycle (new install, write/read, interrupted write, corrupted input, migrations + failed-migration recovery, retention, deletion, export, relaunch, concurrency, uninstall/reset; network/external (offline, DNS/conn failure, timeout, cancel, malformed, auth failure, rate limit, retry limits, partial, server error, privacy-safe logging; deterministic local server/fixtures, no real customer data); reliability (repeated launch/quit, cancellation, interruption/resume, concurrency, duplicates, low-resource, temp cleanup, restart, corrupted state, partial external failure, graceful shutdown); sanitizers/runtime diagnostics (ASan, TSan, UBSan, Main Thread Checker, concurrency diagnostics, leaks, hangs; record unavailable tooling -> blocks the gate; clean normal run != sanitizer run); performance (launch/latency/responsiveness/memory/CPU/disk/energy vs arch budgets; signposts + stable fixtures + recorded env+baseline; no perf claims from debug build); accessibility (keyboard-only, focus order, names/values, menu commands, VoiceOver, contrast/appearance, dynamic text, error announce, permission-denied recovery); release-artifact (exact packaged app, not just dev target: install/deploy, launch w/o debugger, first launch, primary journey smoke, quit+relaunch, persistence, permission timing, bundled resources, final entitlements, privacy manifest, bundled deps, signature, notarization+stapling, Gatekeeper, no debug-only menus/resources, version/build metadata, matching dSYM, clean uninstall).

## Test evidence + coverage + defects
Per run: RUN- {id,purpose,command,redactedArguments[],workingDirectoryRef,startedAt,finishedAt,exitCode,sourceRevision,artifactId,environmentId,testIds[],result,stdoutRef,stderrRef,xcresultRef,coverageRef,attachmentRefs[],evidenceDigest}. Retain+index xcresult. Screenshots alone insufficient when an automatable oracle exists. Don't overwrite a failed bundle with a later success.
Coverage: requirement coverage mandatory (not a decorative global line %). Per MUST: {requirementId,journeyIds[],architectureDecisionIds[],componentIds[],implementationUnitIds[],sourceSymbols[],testIds[],logEventIds[],artifactIds[],evidenceIds[],coverage}. Reverse-validate: every component traced; every dependency traced or platform-justified; every entitlement -> capability; every external service -> data flow; every log event -> diagnostic purpose; every test -> requirement/risk/regression; every file -> implementation unit. Numerical thresholds from arch; regardless, all security/privacy/migration/destructive/permission/error branches tested where reachable.
Defects: BUG- {id,title,severity,discoveredByRunId,affectedRequirements[],affectedComponents[],symptom,expectedBehavior,reproductionSteps[],reproductionEnvironment,operationIds[],logEventIds[],evidenceRefs[],rootCause,regressionTestIds[],correctionImplementationUnits[],verificationRunIds[],status}. Loop: preserve original evidence -> reproduce deterministically -> reduce to smallest reproducer -> trace via operationId/LOG- -> root cause (not symptom) -> regression test that fails first -> smallest arch-compliant fix -> regression passes -> affected tests -> sanitizer/analysis -> full release gate -> closure evidence. Symptom vanishing != fixed. No broad refactor during fix unless root cause requires + separately traced. Intermittent stays open until cause removed + repeated verification stable.

## Security/privacy + dependency verification (verify the BUILT artifact, not source)
Check effective entitlements vs arch; TCC descriptions; sandbox; hardened runtime; runtime exceptions; App Groups; Keychain groups; Apple Events targets; network entitlements; helpers/login items; privacy manifest; required-reason APIs; bundled 3p frameworks; unexpected dylibs; debug entitlements; signing identity + designated requirement; secret/fixture leakage; log privacy; temp cleanup; storage locations; file permissions; transport security; unauthorized network destinations. Any extra entitlement/permission blocks release. Never disable sandbox/hardened runtime to make impl work; insufficient manifest -> architectureIssue, don't expand.
Dependencies: no new runtime dep unless in arch. Dev-only dep only if buildMayDecide permits, cannot ship, pinned, license recorded, source verified, no unauthorized network/data, logged. Verify lock/resolution, versions, checksums, licenses, binary contents, no undeclared transitive runtime deps, deployment-target compat. Never fetch executables from unverified source.

## Build/release protocol + artifact manifest
Exact build model from arch; record all commands: verify toolchain -> resolve pinned deps -> clean build -> full test matrix -> static analysis -> sanitizers -> Release build -> archive -> export/package -> inspect bundle -> extract+compare entitlements -> verify signature -> notarize -> staple+verify -> install exact artifact -> release smoke+acceptance -> compute digests -> preserve symbols -> artifact manifest -> ready gate. Don't sign a different binary than tested; if signing/notarization transforms the artifact, smoke-test the transformed one.
kiln-artifact-manifest.json: ART- {id,type,path,sha256,size,sourceRevision,specRevision,archRevision,devRevision,bundleIdentifier,applicationVersion,buildNumber,supportedArchitectures[],minimumMacOS,buildConfiguration,signingStatus,signingIdentitySummary,notarizationStatus,staplingStatus,effectiveEntitlementsDigest,privacyManifestDigest,bundledDependencies[],binaryUUIDs[],dSYMRefs[],producedByRunIds[],verifiedByRunIds[]}. Never report an uncomputed digest.

## Ready gate (status ready_for_user only when ALL applicable hold)
input gate passed; no intent/architecture issue; no blocking environmentIssue; source revisions match reports exactly; every buildMustImplement complete; every buildMustPreserve preserved; every buildMustNotDo respected; every MUST has complete implementation coverage; every acceptance/unit/integration/UI/system/accessibility/migration/permission-denial/offline-recovery/sanitizer/performance check passed; all evidence retained; no test skipped without non-applicability; no intermittent failure; no known open defect; no production TODO/placeholder; zero unreviewed warnings; no unauthorized dependency/entitlement/endpoint; privacy+secret scans passed; runtime log privacy passed; log catalog consistent; matching dSYM retained; exact release artifact installed+smoke-tested; final signature verified; required notarization succeeded; artifact digests computed; two-way traceability complete; kiln-dev.json / kiln-verification.json / kiln-log-catalog.json / kiln-artifact-manifest.json validate against schema. No kiln:dev-invented waivers.
Final statuses: invalid_input | blocked_on_intent | blocked_on_architecture | blocked_on_environment | implementation_failed | verification_failed | release_blocked | ready_for_user. release_blocked (impl+verification passed but delivery blocked by release-only prerequisite) != ready_for_user.
