# Исполняемый seam-гейт (`kiln check`)

## Контекст
Codex-ревью seam-валидатора (commit `775d8e4`) подтвердил HIGH: `validateArchAgainstIntent` / `validateDevAgainstArch` определены и протестированы, но в рантайме плагина их **ничто не вызывает**. Рантайм Kiln — это проза команд (агент следует markdown); исполняемой точки входа для TS-валидатора не было. Команды ссылались на функцию, которую агент физически не может «вызвать» → шов не enforced.

## Решение
Добавили исполняемый гейт: `src/cli/kiln.ts` с подкомандами `check` (прогоняет cross-stage валидаторы, exit≠0 при нарушении) и `digest` (печатает SHA-256-пин intent-контракта). Раннер — `tsx` (devDep), npm-скрипт `kiln`. Стадии `arch`/`dev`/`pipeline` теперь **запускают** `npm run kiln -- check …` через Bash и падают при ненулевом коде.

## Почему
Весь питч Kiln — валидация «пользователь получил ровно то, что хотел». Валидатор, живущий только в тестах, этому противоречит: в реальном прогоне ничто не остановит `arch`, не сходящийся с `intent`. Исполняемый гейт даёт **машинную гарантию** (а не добросовестность агента) и становится спиной финальной validate-стадии. Цена — рантайм-зависимость `tsx` и то, что плагин теперь «исполняет код», а не только прозу; принято осознанно как ядро продукта (выбор Максима из трёх вариантов).

## Что протестировали
- `kiln digest` печатает ровно те дайджесты, что вшиты в примеры (`c761…`, `63e8…`) → канонический digest стабилен и совпадает с runtime `digestIntent`.
- `kiln check` на согласованном наборе breathing (spec+arch+dev) → `OK`, exit 0.
- `kiln check` breathing-spec против file-renamer-arch → `FAIL digest_mismatch`, exit 1 (гейт реально ловит подмену intent).
- `checkArtifacts` юнит-тесты: ok на примерах, `parse_error` на отсутствующем файле, `digest_mismatch` на кросс-паре. 60/60 зелёные, tsc чист.

## Отвергли
- **Ручные проверки в прозе** — ноль зависимостей, но enforcement зависит от добросовестности агента; не машинная гарантия. Противоречит тезису валидации.
- **Отложить до validate-стадии** — шов остаётся не-enforced, а проза `arch`/`dev` подразумевает вызываемое, которого нет; HIGH не закрыт.
